# Security Policy

## Reporting a Vulnerability

Please **do not open a public GitHub issue** for security vulnerabilities.

Report privately through GitHub's **Private Vulnerability Reporting**:

1. Open the **Security** tab of this repository.
2. Click **Report a vulnerability**.
3. Describe the issue and how to reproduce it.

When reporting, please include where possible:

- The affected version (see the [Releases](https://github.com/Kuro-Boo/KuroCMS/releases) page).
- A clear description and step-by-step reproduction.
- The potential impact, if known.

We aim to acknowledge valid reports within a few business days and will keep you
informed as we work on a fix.

## Supported Versions

KuroCMS is delivered as a rolling release. Only the **latest published release**
receives security fixes. See the
[Releases](https://github.com/Kuro-Boo/KuroCMS/releases) page for the
current version.

## Scope

This repository publishes the **core source of KuroCMS for transparency and
review**. Installation and hosting are performed through the web installer at
<https://kuro.boo/kurocms>; this repository is not intended to be built or
deployed directly.
